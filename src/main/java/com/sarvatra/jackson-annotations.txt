Introduction to Jackson Annotations :-

    1) Jackson is a popular Java library for :-
        - Serializing Java objects into JSON
        - Deserializing JSON into Java objects

    2) By default, Jackson uses JavaBean conventions :-
        - Properties are mapped to JSON fields using getters/setters.
        - Field names are directly used as JSON keys (unless configured otherwise).

    3) But in real-world applications :-
        - Field names in JSON may differ from Java field names.
        - Some fields may need to be ignored.
        - Formatting (e.g., dates) might not match default expectations.
        - Polymorphic types (inheritance) may need explicit hints for correct deserialization.

    ðŸ‘‰ This is where Jackson annotations come in :-
        Jackson annotations are metadata placed on classes, fields, methods, or constructors that tell Jackson how to handle (de)serialization.
    ðŸ‘‰ They live in the package :-
        com.fasterxml.jackson.annotation

1) @JsonProperty :-

    The @JsonProperty annotation in Jackson is used to define the mapping between JSON field names and Java property names
    during serialization (Java â†’ JSON) and deserialization (JSON â†’ Java).

    +---------+----------------+----------------------------------------------------+----------------------------------------------------+
    | Sr. No  |  Property      | Description                                        | Example                                            |
    +---------+----------------+----------------------------------------------------+----------------------------------------------------+
    |   1)    | value          | Name to map in JSON                                | `@JsonProperty("student_id")`                      |
    |   2)    | access         | Controls read/write behavior                       | `READ_ONLY`, `WRITE_ONLY`, `AUTO`, `READ_WRITE`    |
    |   3)    | required       | Marks property as mandatory during deserialization | `@JsonProperty(value="id", required=true)`         |
    |   4)    | defaultValue   | Default value if missing during deserialization    | `@JsonProperty(defaultValue="Unknown")`            |
    |   5)    | index          | Used in array-like serialization                   |  Rarely used in POJOs                              |
    |   6)    | namespace      | For XML (when using Jackson XML module)            | `@JsonProperty(namespace="http://example.com/ns")` |
    +---------+----------------+----------------------------------------------------+----------------------------------------------------+

    +---------+----------------+----------------------+-------------------------------------+-----------------------------------------------+
    | Sr. No  | Property       | Used For             | Example                             | Effect                                        |
    +---------+----------------+----------------------+-------------------------------------+-----------------------------------------------+
    |   1)    | value          | Rename field         | `@JsonProperty("id")`               | Maps JSON name `id` to Java field `studentId` |
    |   2)    | required       | Mandatory field      | `@JsonProperty(required = true)`    | Throws exception if missing                   |
    |   3)    | defaultValue   | Default when missing | `@JsonProperty(defaultValue = "A")` | Fills default on missing JSON field           |
    |   4)    | access         | Control visibility   | `@JsonProperty(access = READ_ONLY)` | Read-only / write-only property               |
    |   5)    | index          | Ordering for arrays  | `@JsonProperty(index = 1)`          | Defines position in array serialization       |
    |   6)    | namespace      | XML namespace        | `@JsonProperty(namespace="...")`    | Used with Jackson XML module                  |
    +---------+----------------+----------------------+-------------------------------------+-----------------------------------------------+
